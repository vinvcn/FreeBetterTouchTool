# Failure Modes

## App lacks permissions

Expected behavior:

- no event tap is created
- menu says permissions missing
- user can open System Settings

## App crashes

Expected behavior:

- process exits
- event tap disappears
- system input returns to normal

## Logi Options+ handles events first

Expected behavior:

- no system damage
- feature may not work as intended
- docs should offer fallback: let ChromeWheelRouter fully own Chrome horizontal scroll in a later mode

## User disables app

Expected behavior:

- all events pass through

## User uninstalls app

Expected behavior:

- no runtime effect remains
- no system-level files remain
